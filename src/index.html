Terminology And Definitions

1. Why is it important to write clean code?
It is important because it needs to be understandable and read-able for yourself or other people to be able to work on the code.
Clean code principles is a foundational skill that pays off especially well when it's time to refactor code or bring code under test.
Developers derive more job satisfaction out of the quality of their work than the quantity.
2. What is the difference between good comments and bad comments?
There are two types of comments, documentation comments and clarification comments.
Bad comments are not clear, too much information, loads of one-line comments, unessasery explination.
3. What is an array?
An array literal is a special type of variable. It stores a list of of values.
They are objects with indexed properties.
4. What are arrays useful for?
Arrays are useful when you want to have a list of options to present to the user or when you want to keep track of all the user input.
They can store many values in one variable.
5. How do you access an array element?
let nameOfArray = ["Root", 55, true, null, undefined];
console.log(nameOfArray[3]);
6. How do you change an array element?
let nameOfArray = ["Root", 55, true, null, undefined];
nameOfArray[3] = "Nothing";
7. What are some useful array properties?
.length
.typeOf()
.indexOf()
.lastIndexOf()
8. What are some useful array methods?
.sort()
.splice()
.pop()
.push()
.concat()
9. What are loops useful for?
Instead of repeating code, loops can execute a block of code a number of times.
Good for running code again and again until it finishes or meets the condition.
10. What is the break statement?
Finishes the condition.
Used to jump out of the loop.
11. What is the continue statement?
Breaks one iteration in the loop if condition is met.
Used to jump over a condition in the loop and continue the loop.
The continue statement breaks one iteration (in the loop), if a specified condition occurs, and continues with the next iteration in the loop.
